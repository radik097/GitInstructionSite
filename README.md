# GitInstructionSite

## Описание сайта

Интерактивный обучающий сайт о том, как пользоваться GitHub. Проект рассчитан на начинающих пользователей и помогает понять базовые термины и рабочий процесс в Git/GitHub через простые объяснения, пошаговые примеры и видео-источники.

## Локализация

Сайт должен поддерживать переключение языков:

- Русский
- Чешский
- Английский

Переключатель языка должен быть доступен на каждой странице и менять весь текст интерфейса (заголовки, кнопки, подсказки, контент уроков).

## Цель

Объяснить пользователю, как безопасно и понятно начать работу с GitHub: от базовых терминов до практических сценариев в GUI и CLI.

## Основные разделы

### 1) Аннотации и термины Git/GitHub

Раздел с раскрывающимися списками (accordion), где каждый термин объясняется простыми словами и с мини-примером.

Минимальный набор терминов:

- Repository (репозиторий)
- Commit
- Branch
- Merge
- Pull Request
- Push
- Pull
- Clone
- Fork
- Conflict

Для каждого термина:

- краткое определение;
- когда используется;
- типичная ошибка новичка;
- маленький пример.

### 2) Примеры использования GUI и CLI

Раздел с двумя режимами обучения:

- GUI (например, GitHub Desktop / интерфейс GitHub)
- CLI (командная строка Git)

В каждом режиме нужно показать порядок действий и объяснить, почему он именно такой.

Пример логики шага:

1. Создать/открыть репозиторий.
2. Изменить файл.
3. Выполнить commit с понятным сообщением.
4. Отправить изменения в удаленный репозиторий (push).
5. Получить изменения с сервера (pull).

Дополнительно:

- блок "Когда лучше GUI, а когда CLI";
- сравнение "быстрее / понятнее / гибче";
- подсказки по типичным ошибкам.

### 3) Источники YouTube

Отдельный блок с проверенными обучающими видео для всех трех языков.

Рекомендуемая структура:

- Русский: 3-5 видео
- Чешский: 3-5 видео
- Английский: 3-5 видео

Для каждого видео:

- название;
- ссылка;
- уровень (Beginner / Intermediate);
- кратко: чему научит.

## Требования к интерактивности

- Раскрывающиеся карточки терминов.
- Переключение языка без перезагрузки страницы (по возможности).
- Прогресс обучения: отметка пройденных шагов.
- Небольшие тест-вопросы после разделов (опционально).

## Ожидаемый результат

Пользователь после прохождения сайта должен:

- понимать базовые термины Git/GitHub;
- знать последовательность работы через GUI и CLI;
- уметь выполнить базовый цикл: изменить файл -> commit -> push -> pull;
- иметь список видео-источников для дальнейшего обучения.
